# 🏢 现代IT公司 — 任务流转图

## 阶段概览
  需求接入 → [👔CTO]
  产品规划 → [📋产品经理]
  技术评审 → [🏗️技术负责人]
  项目排期 → [📊项目经理]
  开发实现 → [🎨前端工程师, ⚙️后端工程师, 🔧DevOps 工程师]
  质量验收 → [🧪QA 工程师]

## 流转步骤

  1. 👔CTO →✅→ 📋产品经理
     创建项目 — 将需求传递给产品经理进行分析

  2. 📋产品经理 →✅→ 🏗️技术负责人
     提交评审 — 将 PRD 和技术方案提交技术评审

  3. 🏗️技术负责人 →🚫→ 📋产品经理
     打回修改 — 技术方案不合格，需要修改
     (可打回至: 📋产品经理)

  4. 🏗️技术负责人 →✅→ 📊项目经理
     评审通过 — 方案可行，进入排期

  5. 📊项目经理 →✅→ frontend_dev + backend_dev + devops
     任务分配 — 将开发任务分配给工程师

  6. frontend_dev + backend_dev + devops →✅→ 📊项目经理
     开发完成 — 提交开发成果

  7. 📊项目经理 →✅→ 🧪QA 工程师
     提测 — 将开发成果推入测试

  8. 🧪QA 工程师 →🚫→ 📊项目经理
     测试未通过 — 发现 Bug 需要修复
     (可打回至: 📊项目经理)

  9. 🧪QA 工程师 →✅→ 👔CTO
     验收通过 — 提交最终交付物

## 审核关卡

  🔍 review 阶段 — 审核人: 🏗️技术负责人
     - 技术方案架构是否合理
     - 技术选型是否合适
     - 是否考虑了扩展性和安全性
     - 任务拆解粒度是否可执行
     封驳打回至: 📋产品经理

  🔍 qa 阶段 — 审核人: 🧪QA 工程师
     - 所有验收标准是否通过
     - 是否存在严重 Bug
     - 性能是否达标
     封驳打回至: 📊项目经理
